第2章 概述

2.1 概述

工汇通系统是针对务工者全过程的管理系统，帮助建设单位对务工者进行有效的管理和及时跟踪务工者的具体情况，打通了业主单位跟务工者之间的桥梁。

2.2 项目背景

我国农民工工资拖欠问题一直存在，特别是建设领域用工管理和工资支付行为不规范，导致务工者工资拖欠问题尤为严重。目前，国务院办公厅印发《关于全面治理拖欠务工者工资问题的意见》，就健全预防和解决拖欠务工者工资问题的长效机制，从根本上治理拖欠务工者工资问题提出了明确要求。

2.3 建设目标

建立完善的务工者管理体系，建立个人档案，对务工者的进场、培训、发放工资、离场、信用评价等全生命周期进行有效的跟踪记录和实时反馈。

第3章 需求分析

3.1 模块需求

建立工队及务工者进场、培训、工资发放、信用评价等模块。

3.2 接口需求

开发适用于钉钉端、安卓端、IOS端的数据接口，包括汇总统计，数据分析，各大模块数据列表等。

3.3 性能需求

适用于大数据高并发，读写分离高效系统。

3.4安全需求

服务器安全，网站安全，程序安全，数据资料安全。

第4章 系统配置

4.1 服务器

Windows Server 2008 R2 Standard

4.2 数据库

Mysql version 5.7.13

4.3 软件框架

ASP.NET MVC 4.5

4.4 开发工具

Visual Studio 2013

第5章 系统架构

5.1 系统架构图


工汇通系统采用典型的MVC架构，主要分成应用层、业务层和控制层。根据功能划分为客户端应用、业务层、数据层、基础服务层及系统运行环境。

5.2 角色功能图


工汇通系统采用角色功能来分配任务并进行权限控制，利于不同人员的职责和分工明确，提高效率。

第6章 项目实施

6.1 建立运行管理机制

由业主单位主推，施工单位实施并对过程跟踪反馈，建立由上而下的一套协同运转机制。

6.2 人员安排

业主单位安排专门管理人员，项目部安排专管员，负责数据的收集和录入。

6.2 系统试运行

工汇通系统将在佛山地铁2号线一期工程中，对每个施工单位逐步推行试运行，并及时收集反馈意见对系统进行完善更新。

6.2 评审制度

业主单位对工汇通推广和运行情况会阶段性进行检查，对运行情况良好的进行表彰，运行较差的进行通报批评。
